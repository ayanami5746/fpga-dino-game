# 数逻final-dino分工  

## 一、分工预览  
| 游戏逻辑层（成员A） | 前端显示层（成员B） |
|---------------------|---------------------|
| 只负责**状态计算**，不产生任何像素数据 | 只负责**像素渲染**，不修改任何游戏状态 |
| 输入：键盘、时钟、复位 | 输入：游戏逻辑输出的所有状态信号 |
| 输出：所有元素的坐标、类型、游戏状态、音效触发 | 输出：VGA信号、帧同步信号 |
| 所有计算基于**整数坐标**，与像素无关 | 所有渲染基于**像素坐标**，与游戏逻辑无关 |

## 二、坐标与像素约定
**所有元素的坐标均为元素**`左上角`**的像素坐标**。
- 屏幕坐标系：640×480像素，左上角为(0,0)，向右为X轴正方向，向下为Y轴正方向
- 元素大小：所有元素的宽高为`200-offline-sprite.png`图片中的0.75倍，如恐龙大小为66×70像素（图片中为88×94像素）
- 固定坐标：恐龙X坐标固定为0，地面Y坐标固定为360，Y坐标低于100的部分留作空白

## 三、具体分工
### 成员A：游戏逻辑（6个核心模块）
**核心职责**：处理输入，计算所有游戏状态，输出状态信号
**具体任务**：
1. **PS/2键盘输入处理**
   - 实现PS/2协议解码和按键消抖
   - 检测键盘的按下和释放事件
   - 输出单周期的`jump_trigger`信号
2. **游戏主状态机**
   - 实现4种游戏状态：待机、游戏中、游戏结束（*、暂停*）
   - 处理状态转换逻辑
   - 输出全局`game_state[1:0]`信号
3. **物理引擎与元素移动系统**
   - 实现恐龙的跳跃物理（重力加速度、初速度、落地检测）
   - 实现恐龙跑步动画切换（根据游戏速度调整切换频率）
   - 实现仙人掌、翼龙、云朵等的匀速向左移动
   - 实现游戏速度随分数递增逻辑
4. **随机元素生成系统**
   - 实现伪随机数生成器
   - 随机生成仙人掌（小/大）和翼龙
   - 随机控制元素生成间隔
   - 实现昼夜周期计时和月相变化
5. **碰撞检测系统**
   - 实现恐龙与仙人掌的矩形碰撞检测
   - 实现恐龙与翼龙的矩形碰撞检测
   - 实现恐龙与地面的碰撞检测
   - 碰撞发生时触发游戏结束状态
6. **分数系统**
   - 实现实时分数计算
   - 实现分数的BCD码转换（个位、十位、百位、千位、万位）
   - 实现每100积分触发一次蜂鸣逻辑
7. **顶层逻辑模块封装**
   - 将所有显示子模块封装为一个`game_logic.v`模块
   - 提供约定的的输入输出接口
   - 编写显示系统的单元测试bench

### 成员B：前端显示（6个核心模块）
**核心职责**：将游戏逻辑输出的状态信号转换为VGA像素信号，实现视听觉特效
**具体任务**：
1. **VGA控制器与ROM资源库**
   - 维护并优化`vga_ctrl.v`代码
   - 制作包含所有资源的ROM IP核：恐龙姿态、仙人掌、翼龙、云朵、数字0-9、地面、Gameover图标、星星、月亮及月相
   - 定义并维护ROM地址映射表
2. **像素生成器**
   - 实现优先级渲染：Gameover > 计分板 > 恐龙 > 仙人掌 > 翼龙 > 月亮 > 云朵 > 地面
   - 实现所有元素的区域判断和ROM地址计算
   - 处理透明像素
3. **昼夜与月相变化系统**
   - 接收游戏逻辑输出的`night`信号和`day_night_cycle[3:0]`信号
   - 实现背景色从白天到黑夜、从黑夜到白天的平滑过渡
   - 根据`moon_phase[2:0]`信号显示对应月相的月亮
   - 夜晚阶段，所有像素取反色
4. **Gameover界面系统**
   - 接收`game_state`信号，当为"游戏结束"时显示Gameover图标和Restart图标
   - 实现游戏结束时Restart图标的变换
5. **时钟系统控制**
   - 将SWORD的100MHz的时钟转化为适用于VGA的25MHz的时钟
   - 生成帧同步信号frame_end
6. **音效系统**
   - 实现跳跃音效、碰撞音效、得分音效
   - 控制蜂鸣器输出
7. **顶层显示模块封装**
   - 将所有显示子模块封装为一个`display_system.v`模块
   - 提供约定的的输入输出接口
   - 编写显示系统的单元测试bench


## 四、接口约定
时钟信号为SWORD板提供的**100MHz信号**，高电平有效。

### 全局信号
| 信号名 | 方向 | 位宽 | 说明 |
|--------|------|------|------|
| clk | 输入 | 1 | 100MHz全局时钟 |
| rst | 输入 | 1 | 全局异步高电平复位 |

### A → B：游戏状态信号
所有坐标均为元素左上角像素坐标。
| 信号名 | 位宽 | 说明 |
|--------|------|------|
| game_state | 2 | 00=待机，01=游戏中，11=游戏结束（*，10=暂停*） |
| dino_y | 9 | 恐龙左上角像素的Y坐标（X固定为0） |
| dino_state | 3 | 000=静止/跳姿，001=跑姿1，011=跑姿2，010=蹲姿1，110=蹲姿2，111=死亡姿态 |
| ground_x0 | 11 | 有符号数（signed），表示0号地面左上角像素的X坐标（Y固定为360） |
| ground_x1 | 11 | 有符号数（signed），表示1号地面左上角像素的X坐标（Y固定为360） |
| cactus_valid | 3 | 仙人掌的3个槽位是否有效 |
| cactus_x0 | 10 | 0号仙人掌左上角X坐标 |
| cactus_x1 | 10 | 1号仙人掌左上角X坐标 |
| cactus_x2 | 10 | 2号仙人掌左上角X坐标 |
| cactus_type0 | 3 | 0号仙人掌类型：000=无仙人掌，001:011=小仙人掌，100:110=大仙人掌 |
| cactus_type1 | 3 | 1号仙人掌类型：000=无仙人掌，001:011=小仙人掌，100:110=大仙人掌 |
| cactus_type2 | 3 | 2号仙人掌类型：000=无仙人掌，001:011=小仙人掌，100:110=大仙人掌 |
| pterodactyl_valid | 2 | 翼龙的2个槽位是否有效|
| pterodactyl_x0 | 10 | 0号翼龙左上角X坐标 |
| pterodactyl_x1 | 10 | 1号翼龙左上角X坐标 |
| pterodactyl_y0 | 9 | 0号翼龙左上角Y坐标 |
| pterodactyl_y1 | 9 | 1号翼龙左上角Y坐标 |
| pterodactyl_state0 | 1 | 0号翼龙类型：0=翅膀上，1=翅膀下 |
| pterodactyl_state1 | 1 | 1号翼龙类型：0=翅膀上，1=翅膀下 |
| cloud_valid | 3 | 云朵的3个槽位是否有效 |
| cloud_x0 | 10 | 0号云朵左上角X坐标 |
| cloud_x1 | 10 | 1号云朵左上角X坐标 |
| cloud_x2 | 10 | 2号云朵左上角X坐标 |
| cloud_y0 | 9 | 0号云朵左上角Y坐标 |
| cloud_y1 | 9 | 1号云朵左上角Y坐标 |
| cloud_y2 | 9 | 2号云朵左上角Y坐标 |
| moon_phase | 3 | 0-5表示6种月相 |
| night| 1 | 0表示白天，1表示夜晚 |
| day_night_cycle | 3 | 0-7表示8个昼夜切换阶段（为平滑过渡） |
| sound_press | 1 | 1表示需要播放跳跃音效 |
| sound_hit | 1 | 1表示需要播放游戏结束音效 |
| sound_reached | 1 | 1表示需要播放得分到达整百的音效 |
| score0 | 4 | 分数个位（BCD） |
| score1 | 4 | 分数十位（BCD） |
| score2 | 4 | 分数百位（BCD） |
| score3 | 4 | 分数千位（BCD） |
| score4 | 4 | 分数万位（BCD） |

### B → A：同步信号
| 信号名 | 位宽 | 说明 |
|--------|------|------|
| frame_end | 1 | 每帧（1/60秒）输出一个时钟周期的高脉冲<br>**游戏逻辑所有状态更新必须在这个信号的上升沿触发** |
| clk_25m | 1 | 用于VGA的25MHz时钟 |
